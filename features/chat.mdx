Chat is part of the [Query](/features/query) workflow. It runs retrieval and answer generation together, with conversation history.

## Interface Docs

Use the interface-specific docs when you need exact parameters, flags, request fields, or response shapes.

- [SDK chat](/usage/sdk/chat)
- [CLI chat](/usage/cli/chat)
- [REST chat](/usage/server/chat)

<CodeGroup>
```python SDK
result = rag.chat("How do I authenticate?", history=[])
print(result.answer)

for event in rag.chat_stream("How do I authenticate?", history=[]):
    if event["type"] == "token":
        print(event["data"]["text"], end="")
```

```bash CLI
ragrails chat "How do I authenticate?" \
  --vector-db qdrant --collection docs --url http://localhost:6333 \
  --embedder-provider voyage --embedder-model voyage-3 \
  --llm-provider openai --llm-model gpt-4o-mini
```

```bash REST API
curl -X POST http://127.0.0.1:8000/v1/chat \
  -H "Content-Type: application/json" \
  -d '{
    "query": "How do I authenticate?",
    "llm_provider": "openai",
    "llm_model": "gpt-4o-mini",
    "embedder_provider": "voyage",
    "embedder_model": "voyage-3",
    "vector_db": "qdrant",
    "collection": "docs",
    "url": "http://localhost:6333",
    "history": []
  }'

curl -N -X POST http://127.0.0.1:8000/v1/chat/stream \
  -H "Content-Type: application/json" \
  -d '{"query":"How do I authenticate?","collection":"docs"}'
```
</CodeGroup>

<Note>Streaming emits progress, token, error, and final events for normal RAG chat. Agentic tool-call streaming is not currently covered.</Note>

<Note>**Chat is stateless.** Ragrails never stores conversations. You pass `history` in and get the updated `history` back, so you decide where it lives.</Note>

## Tuning knobs

| Symptom | Config | What it does |
| --- | --- | --- |
| Follow-ups miss context | `QueryRewriteConfig` | Expands follow-ups into standalone questions. |
| Long chats slow / costly | `HistoryCompactionConfig` | Summarizes old turns, keeps recent ones. |
| Small talk triggers search | `IntentRoutingConfig` | Skips retrieval for non-questions. |
| Model answers with weak context | `ChatRetrievalQualityConfig` | Sets confidence thresholds; answers with caution or refuses. |

### Query rewriting

<CodeGroup>
```python SDK
from ragrails import QueryRewriteConfig
rag.chat("What about the second step?", history=history, query_rewrite=QueryRewriteConfig(enabled=True))
```

### Retrieval quality

<CodeGroup>
```python SDK
from ragrails import ChatRetrievalQualityConfig
rag.chat(
    query,
    history=history,
    retrieval_quality=ChatRetrievalQualityConfig(
        min_retrieval_score=0.35,
        low_confidence_mode="answer_with_caution",
    ),
)
```

```bash CLI
ragrails chat "$QUERY" \
  --vector-db qdrant --collection docs --url http://localhost:6333 \
  --embedder-provider voyage --embedder-model voyage-3 \
  --llm-provider openai --llm-model gpt-4o-mini
```

```bash REST API
curl -X POST http://127.0.0.1:8000/v1/chat \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Can I get a refund after 30 days?",
    "retrieval_quality": {"min_retrieval_score": 0.35, "low_confidence_mode": "answer_with_caution"},
    "llm_provider": "openai",
    "llm_model": "gpt-4o-mini",
    "embedder_provider": "voyage",
    "embedder_model": "voyage-3",
    "vector_db": "qdrant",
    "collection": "docs",
    "url": "http://localhost:6333",
    "history": []
  }'
```
</CodeGroup>

<CodeGroup>

<Note>The CLI does not expose retrieval-quality thresholds yet; use SDK or REST when you need those controls.</Note>

## What you get back

| Field | Use it for |
| --- | --- |
| `answer` | The grounded reply |
| `history` | Pass to the next turn |
| `sources` | Show citations |
| `intent` | `"rag"` or `"direct"` |
| `answer_confidence` | Flag low-confidence answers in your UI |
