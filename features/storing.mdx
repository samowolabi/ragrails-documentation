---
title: "Storing"
description: "Store chunks in a vector database."
---

Storing is the explicit vector database write stage. It uses the same provider arguments as embedding.

## Capabilities

- Use a storage-oriented `store` method, command, endpoint, or MCP tool.
- Embed chunk files and write vectors to a vector database in one stage.
- Store in Qdrant, Pinecone, or Weaviate.
- Validate provider-specific collection/index/class naming rules.
- Process selected chunk JSON files or every JSON file in a chunk directory.

## Relationship to embedding

`store` uses the same underlying embedding-and-upsert path as `embed`. Use
`store` when you want the pipeline to read clearly as ingestion -> chunking ->
storing -> retrieval.
