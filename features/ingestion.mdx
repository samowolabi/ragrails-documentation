---
title: "Ingestion"
description: "Convert URLs, documents, and REST API responses into markdown."
---

Ingestion creates markdown files with source metadata. Chunking consumes these markdown files in the next stage.

## Capabilities

- Scrape one or more exact URLs.
- Crawl full websites with depth and page limits.
- Retry failed URL ingestion from a dead-letter queue.
- Parse local documents into markdown.
- Fetch REST API responses with methods, headers, query params, JSON bodies, pagination, and max page limits.
- Write markdown with source metadata for downstream chunking.

## Supported document types

Folder-based document ingestion supports common document and data formats:

```text
.csv, .docx, .epub, .html, .htm, .ipynb, .json, .md, .msg,
.pdf, .pptx, .rss, .tsv, .txt, .xls, .xlsx, .xml, .zip
```

    ```bash
    curl -X POST http://127.0.0.1:8000/v1/ingest/docs \
      -H "Content-Type: application/json" \
      -d '{"folder":"files/input","output_dir":"files/output/docs"}'

    curl -X POST http://127.0.0.1:8000/v1/ingest/api \
      -H "Content-Type: application/json" \
      -d '{"url":"https://api.example.com/v1/products","title":"Products","output_dir":"files/output/api"}'
    ```

    URL ingestion endpoint:

    ```text
    POST /v1/ingest/url
    ```
