## The problem

Shoppers describe what they want in their own words, like *"a waterproof jacket for hiking under $200"*, but keyword filters can't parse intent.

## How Ragrails solves it

Pull your product catalog from its API, embed the product descriptions, and let shoppers search or chat in natural language.

1. **Fetch** the catalog API (paginated) into a collection.
2. **Retrieve or chat** over product descriptions and attributes.

<CodeGroup>
```python SDK
from ragrails import RagRails

rag = RagRails(
    collection="products",
    vector_store={"provider": "qdrant", "url": "http://localhost:6333"},
    embedding={"provider": "voyage", "model": "voyage-3"},
    llm={"provider": "openai", "model": "gpt-4o-mini"},
)
rag.ingest(api={
    "url": "https://api.shop.example.com/products",
    "title": "Products",
    "pagination": {"type": "page", "param": "page", "size_param": "per_page", "size": 100},
    "max_pages": 50,
})
hits = rag.retrieve("waterproof hiking jacket under $200", top_k=10)
```

```bash CLI
ragrails ingest \
  --api-url https://api.shop.example.com/products \
  --vector-db qdrant --collection products --url http://localhost:6333 \
  --provider voyage --model voyage-3

ragrails retrieve "waterproof hiking jacket under $200" \
  --vector-db qdrant --collection products --url http://localhost:6333 \
  --provider voyage --model voyage-3 --top-k 10
```

```bash REST API
curl -X POST http://127.0.0.1:8000/v1/pipelines/ingest \
  -H "Content-Type: application/json" \
  -d '{"api": {"url": "https://api.shop.example.com/products", "title": "Products", "pagination": {"type": "page", "param": "page", "size_param": "per_page", "size": 100}, "max_pages": 50}, "embedding": {"provider": "voyage", "model": "voyage-3"}, "storage": {"vector_db": "qdrant", "collection": "products", "url": "http://localhost:6333"}}' 

curl -X POST http://127.0.0.1:8000/v1/retrieve \
  -H "Content-Type: application/json" \
  -d '{"query": "waterproof hiking jacket under $200", "provider": "voyage", "model": "voyage-3", "vector_db": "qdrant", "collection": "products", "url": "http://localhost:6333", "top_k": 10}' 
```
</CodeGroup>

<Tip>Refresh the catalog regularly with [`edit()`/`delete()`](/features/knowledge-base-maintenance) so discontinued products drop out of results.</Tip>
