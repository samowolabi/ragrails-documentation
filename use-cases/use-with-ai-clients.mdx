---
title: "Use with AI Clients"
description: "Expose Ragrails as local MCP tools for AI clients and agents."
---

Use the MCP server when you want an AI client to call Ragrails tools directly
from your machine.

## Good fits

- Let an AI client parse local documents into markdown.
- Let an AI client chunk a prepared folder.
- Let an AI client retrieve context from a local or private vector database.
- Keep files, credentials, and vector database access local.

## Usage

<Tabs>
  <Tab title="MCP">
    Configure your AI client:

    ```json
    {
      "mcpServers": {
        "ragrails": {
          "command": "ragrails-mcp"
        }
      }
    }
    ```

    Example requests:

    ```text
    Use Ragrails to parse files/input into files/output/docs.
    ```

    ```text
    Use Ragrails to retrieve "How do refunds work?" from the qdrant collection docs.
    ```
  </Tab>

  <Tab title="SDK">
    Use the SDK when you are building the agent runtime in Python:

    ```python
    from ragrails import RagRails

    result = RagRails().retrieve(
        query="How do refunds work?",
        vector_db="qdrant",
        collection="docs",
        top_k=10,
    )
    ```
  </Tab>

  <Tab title="REST API">
    Use the REST API when your AI client or orchestration layer calls HTTP tools:

    ```bash
    curl -X POST http://127.0.0.1:8000/v1/retrieve \
      -H "Content-Type: application/json" \
      -d '{
        "query": "How do refunds work?",
        "vector_db": "qdrant",
        "collection": "docs",
        "top_k": 10
      }'
    ```
  </Tab>
</Tabs>
