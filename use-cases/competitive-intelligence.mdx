## The problem

Competitors update pricing, features, and positioning constantly. Manually checking their sites doesn't scale, and the info goes stale fast.

## How Ragrails solves it

Crawl competitor sites and news on a schedule, recover failures automatically, and keep the index fresh.

1. **Full-crawl** competitor sites with a [dead-letter queue](/advanced/resilient-ingestion) so partial failures are recoverable.
2. **Refresh** on a schedule: re-ingest, and `delete()` pages that disappeared.
3. **Query or chat** to compare and summarize.

<CodeGroup>
```python SDK
from ragrails import RagRails

rag = RagRails(
    collection="competitors",
    vector_store={"provider": "qdrant", "url": "http://localhost:6333"},
    embedding={"provider": "voyage", "model": "voyage-3"},
    llm={"provider": "openai", "model": "gpt-4o-mini"},
)
rag.setup_url()
rag.ingest(urls=["https://competitor-a.com", "https://competitor-b.com"], ingestion={"urls": {"mode": "full", "max_pages": 200}})
rag.query("What changed in competitor pricing?", retrieval={"top_k": 5})
```

```bash CLI
ragrails setup-url
ragrails ingest \
  --source-url https://competitor-a.com \
  --source-url https://competitor-b.com \
  --vector-db qdrant --collection competitors --url http://localhost:6333 \
  --provider voyage --model voyage-3

ragrails query "What changed in competitor pricing?" \
  --vector-db qdrant --collection competitors --url http://localhost:6333 \
  --provider voyage --model voyage-3 --top-k 5
```

```bash REST API
curl -X POST http://127.0.0.1:8000/v1/pipelines/ingest \
  -H "Content-Type: application/json" \
  -d '{"urls": ["https://competitor-a.com", "https://competitor-b.com"], "ingestion": {"urls": {"mode": "full", "max_pages": 200}}, "embedding": {"provider": "voyage", "model": "voyage-3"}, "storage": {"vector_db": "qdrant", "collection": "competitors", "url": "http://localhost:6333"}}' 

curl -X POST http://127.0.0.1:8000/v1/pipelines/query \
  -H "Content-Type: application/json" \
  -d '{"query": "What changed in competitor pricing?", "embedding": {"provider": "voyage", "model": "voyage-3"}, "retrieval": {"vector_db": "qdrant", "collection": "competitors", "url": "http://localhost:6333", "top_k": 5}}' 
```
</CodeGroup>

<Tip>Schedule the crawl (cron, a worker) and use [`edit()`/`delete()`](/features/knowledge-base-maintenance) so answers always reflect the latest competitor state.</Tip>
