---
title: "CLI Embedding"
icon: "binary"
description: "Embed chunk JSON files from the terminal."
---

`embed` reads chunk JSON from `--input-dir`, generates vectors, and writes an `embedded.json` file to `--output-dir`. Embedding is separate from storage; pass the output to `store` next.

```bash
pip install "ragrails[voyage]"
```

<CodeGroup>
```bash Command
ragrails embed \
  --input-dir files/chunks/ \
  --output-dir files/embedded/ \
  --provider voyage \
  --model voyage-3 \
  --batch-size 64
```

```text Output
Inputs   : 87
Embedded : 87
Failed   : 0
Saved to : files/embedded/embedded.json
```
</CodeGroup>

The CLI embeds with `input_type="document"` (for indexing). Query-time embedding happens inside `retrieve`.

<Tip>Use the same provider and model for embedding and retrieval. Vectors from different models aren't comparable. Raise `--batch-size` to cut API calls, lower it if you hit rate limits.</Tip>

## Options

| Option | Default | Description |
| --- | --- | --- |
| `--input-dir` | required | Folder of chunk JSON files |
| `--output-dir` | required | Folder to write `embedded.json` to |
| `--provider` | `voyage` | Embedding provider |
| `--model` | `voyage-3` | Embedding model |
| `--batch-size` | `64` | Chunks per embedding request |
