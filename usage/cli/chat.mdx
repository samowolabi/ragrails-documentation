`chat` answers a question grounded in your vector index. Pass a query for a one-shot turn, or run it with no query to start an interactive session.

## One-shot chat turn

<CodeGroup>
```bash Command
ragrails chat "How do I authenticate?" \
  --vector-db qdrant \
  --collection docs \
  --url http://localhost:6333 \
  --llm-provider openai \
  --llm-model gpt-4o-mini
```

```text Output
To authenticate, create an API key in your account settings...

Sources:
[1] Authentication
```
</CodeGroup>

## Multi-turn with a history file

`--history-file` persists conversation history across turns. The file is created on the first turn and updated after each one.

```bash
ragrails chat "How do I authenticate?" \
  --history-file files/chat/history.json \
  --vector-db qdrant --collection docs --url http://localhost:6333 \
  --llm-provider openai --llm-model gpt-4o-mini

ragrails chat "What about the second step?" \
  --history-file files/chat/history.json --rewrite-query
```

## Interactive session

Run `chat` with no query to start a REPL that keeps history, streams answers, and can call tools.

```bash
ragrails chat
```

### Slash commands

| Command | What it does |
| --- | --- |
| `/model` | List models, or `/model gpt-4o` to switch mid-conversation |
| `/stream on` / `/stream off` | Toggle token-by-token streaming |
| `/tools` | List the tools the assistant can call |
| `/debug` | Show the retrieval trace for the last answer |

### Agentic tools

When the answer isn't in your index, the assistant can call `web_fetch` (read a web page) or `api_call` (make a live HTTP request). It confirms before each call. See [agentic tools](/advanced/agentic-tools).

## Flags map to SDK config

| Flag | SDK config |
| --- | --- |
| `--rewrite-query`, `--rewrite-session-context` | `QueryRewriteConfig` |
| `--disable-history-compaction` | `HistoryCompactionConfig(enabled=False)` |
| `--disable-intent-routing` | `IntentRoutingConfig(enabled=False)` |
| `--rerank`, `--reranker`, `--reranker-model`, `--rerank-top-k` | reranker |

## Options

| Option | Default | Description |
| --- | --- | --- |
| `QUERY` | none | Question (positional). Omit to start the interactive REPL |
| `--llm-provider` | `openai` | LLM provider (`openai`, `anthropic`, `google`) |
| `--llm-model` | `gpt-4o-mini` | LLM model |
| `--max-tokens` | `1024` | Max output tokens |
| `--embedder-provider` | `voyage` | Query embedding provider |
| `--embedder-model` | `voyage-3` | Query embedding model |
| `--vector-db` | `qdrant` | Vector DB provider |
| `--collection` | none | Collection name |
| `--url` | none | Vector DB URL |
| `--persona` | `""` | System persona for the turn |
| `--history-file` | none | JSON file to load and save history |
| `--rewrite-query` | off | Rewrite the query before retrieval |
| `--rewrite-session-context` | `""` | Context hint for query rewriting |
| `--rerank` | off | Rerank retrieved chunks |
| `--reranker` / `--reranker-model` / `--rerank-top-k` | `voyage` / `rerank-2-lite` / `5` | Reranker config |
| `--disable-intent-routing` | off | Always run retrieval, even for small talk |
| `--disable-history-compaction` | off | Return full history without summarisation |
