---

`chunk` reads ingestion-output JSON from `--input-dir` and writes a `chunks.json` file to `--output-dir`. Chunking splits documents into focused passages so search can match the exact paragraph that answers a query.

<CodeGroup>
```bash Command
ragrails chunk \
  --input-dir files/output/docs/ \
  --output-dir files/chunks/ \
  --chunk-size 2000 \
  --chunk-overlap 200 \
  --min-chunk-length 100
```

```text Output
Documents : 12
Chunks    : 87
Failed    : 0
Saved to  : files/chunks/chunks.json
```
</CodeGroup>

## Choosing chunk size

`--chunk-size` is the main lever. Too small loses surrounding context; too large retrieves noise around the answer. Start at `2000` with `200` overlap (~10%), lower the size for dense factual content like specs or FAQs.

## Options

| Option | Default | Description |
| --- | --- | --- |
| `--input-dir` | required | Folder of ingestion output JSON files |
| `--output-dir` | required | Folder to write `chunks.json` to |
| `--chunk-size` | `2000` | Max characters per chunk |
| `--chunk-overlap` | `200` | Character overlap between chunks |
| `--min-chunk-length` | `100` | Drop chunks shorter than this |
