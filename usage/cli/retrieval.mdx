title: "CLI Retrieval"

description: "Search stored chunks from the terminal."
---

`retrieve` embeds the query, searches the vector store, and prints the most relevant chunks. Add `--rerank` to reorder the top candidates by true relevance.

<CodeGroup>
```bash Command
ragrails retrieve "How do I authenticate?" \
  --vector-db qdrant \
  --collection docs \
  --url http://localhost:6333 \
  --provider voyage \
  --model voyage-3 \
  --top-k 10
```

```text Output
Query  : How do I authenticate?
Results: 10

[1] score=0.8700 id=guide-3
    title: Authentication
    text : To authenticate, create an API key in settings...
```
</CodeGroup>

## With reranking

Retrieve wide, rerank down. The reranker needs candidates to choose from, so keep `--top-k` higher than `--rerank-top-k`.

<CodeGroup>
```bash Command
ragrails retrieve "How do I authenticate?" \
  --vector-db qdrant --collection docs --url http://localhost:6333 \
  --provider voyage --model voyage-3 \
  --top-k 20 \
  --rerank --reranker voyage --reranker-model rerank-2-lite --rerank-top-k 5
```

```text Output
Query  : How do I authenticate?
Results: 5

[1] score=0.8100 rerank=0.9300 id=guide-3
    title: Authentication
    text : To authenticate, create an API key in settings...
```
</CodeGroup>

## Options

| Option | Default | Description |
| --- | --- | --- |
| `QUERY` | required | Search query (positional) |
| `--vector-db` | `qdrant` | `qdrant`, `pinecone`, or `weaviate` |
| `--collection` | none | Collection name |
| `--url` | none | Vector DB URL |
| `--top-k` | `10` | Results to retrieve |
| `--provider` | `voyage` | Query embedding provider |
| `--model` | `voyage-3` | Query embedding model |
| `--rerank` | off | Enable reranking |
| `--reranker` | `voyage` | Reranker provider (`voyage` or `bm25`) |
| `--reranker-model` | `rerank-2-lite` | Reranker model |
| `--rerank-top-k` | `5` | Results to keep after reranking |

<Note>Query rewriting isn't exposed on the `retrieve` command. Use it in `ragrails chat --rewrite-query`, or the [SDK `retrieve()`](/usage/sdk/retrieval#with-query-rewriting) / [REST](/usage/server/retrieval#with-query-rewriting).</Note>
