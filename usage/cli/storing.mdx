Three commands manage chunks in a vector database. The collection is created automatically if it doesn't exist.

## Store (`store`)

Reads embedded chunk JSON from `--input-dir` and upserts it.

<CodeGroup>
```bash Command
ragrails store \
  --input-dir files/embedded/ \
  --vector-db qdrant \
  --collection docs \
  --url http://localhost:6333 \
  --batch-size 64
```

```text Output
Inputs     : 12
Stored     : 12
Failed     : 0
Provider   : qdrant
Collection : docs
```
</CodeGroup>

| Option | Default | Description |
| --- | --- | --- |
| `--input-dir` | required | Folder of embedded chunk JSON files |
| `--vector-db` | `qdrant` | `qdrant`, `pinecone`, or `weaviate` |
| `--collection` | none | Collection name |
| `--url` | none | Vector DB URL |
| `--batch-size` | `64` | Chunks per upsert batch |

## Edit (`edit`)

Re-embeds and replaces stored chunks. Files must contain chunk dicts with matching IDs.

<CodeGroup>
```bash Command
ragrails edit \
  --input-dir files/updated/ \
  --vector-db qdrant --collection docs --url http://localhost:6333 \
  --provider voyage --model voyage-3
```

```text Output
Requested  : 1
Edited     : 1
Failed     : 0
Provider   : qdrant
Collection : docs
```
</CodeGroup>

## Delete (`delete`)

Removes chunks by ID.

<CodeGroup>
```bash Command
ragrails delete \
  --id guide-0 --id guide-1 \
  --vector-db qdrant --collection docs --url http://localhost:6333
```

```text Output
Requested  : 2
Deleted    : 2
Failed     : 0
Provider   : qdrant
Collection : docs
```
</CodeGroup>

<Tip>Give chunks stable IDs at chunk time so `edit` and `delete` can target them. See [keeping your index current](/features/knowledge-base-maintenance).</Tip>
