title: "SDK Storing"
icon: "database"
description: "Store, edit, and delete chunks in a vector database."
---

Storing upserts embedded chunks into a vector database. With a vector store configured on `RagRails`, you don't repeat the connection on every call. The collection is created automatically if it doesn't exist.

```python
from ragrails import RagRails

rag = RagRails(
    collection="docs",
    vector_store={"provider": "qdrant", "url": "http://localhost:6333"},
    embedding={"provider": "voyage", "model": "voyage-3"},
)
```

<Note>The constructor takes `vector_store={"provider": ...}` plus `collection=`. The `store`, `edit`, and `delete` methods take `vector_db=`, `collection=`, and `url=` to override.</Note>

## Store

<CodeGroup>
```python Request
result = rag.store(embedded_chunks=embed_result.items)
```

```python Response
result.stored      # 12
result.failed      # 0
result.provider    # "qdrant"
result.collection  # "docs"
```
</CodeGroup>

| Parameter | Default | Description |
| --- | --- | --- |
| `embedded_chunks` | required | Chunk dicts with `id`, `text`, `metadata`, `embedding` |
| `vector_db` | inherits constructor | `"qdrant"`, `"qdrant_cloud"`, `"pinecone"`, `"weaviate"` |
| `collection` | inherits constructor | Collection, index, or class name |
| `url` | inherits constructor | Vector DB URL |
| `batch_size` | `64` | Chunks per upsert request |
| `ensure_collection` | `True` | Create the collection if missing |
| `options` | `None` | Provider-specific options |

## Edit

Re-embed and replace stored chunks by ID. The embedder is created from the constructor `embedding` config when omitted.

<CodeGroup>
```python Request
result = rag.edit(chunks=[
    {"id": "guide-0", "text": "Refunds now take 3 to 5 business days.", "source": "files/guide.pdf", "metadata": {"title": "Refunds"}},
])
```

```python Response
result.requested  # 1
result.edited     # 1
result.failed     # 0
```
</CodeGroup>

## Delete

```python
result = rag.delete(ids=["guide-0", "guide-1"])
result.deleted  # 2
```

## Supported databases

| Database | `vector_db` | Notes |
| --- | --- | --- |
| Qdrant (local) | `qdrant` | Needs `url` (default `http://localhost:6333`) |
| Qdrant Cloud | `qdrant_cloud` | Cloud `url` + `QDRANT_API_KEY` |
| Pinecone | `pinecone` | Uses `PINECONE_API_KEY` |
| Weaviate | `weaviate` | Local or cloud, needs `url` |

<Tip>Give chunks stable IDs at chunk time so `edit` and `delete` can target them. See [keeping your index current](/features/knowledge-base-maintenance).</Tip>
