Retrieval embeds the query, searches the vector store, and returns the closest chunks. With providers configured on `RagRails`, `retrieve()` is a one-liner; it creates the query embedder for you.

```python
from ragrails import RagRails

rag = RagRails(
    collection="docs",
    vector_store={"provider": "qdrant", "url": "http://localhost:6333"},
    embedding={"provider": "voyage", "model": "voyage-3"},
    reranker={"provider": "voyage", "model": "rerank-2-lite"},
)
```

<CodeGroup>
```python Request
result = rag.retrieve("How do I authenticate?", top_k=10)

for chunk in result.items:
    print(chunk.score, chunk.text)
```

```python Response
result.retrieved  # 10
result.items[0]   # RetrievedChunk(
                  #   id="guide-3",
                  #   score=0.87,
                  #   text="To authenticate, create an API key in settings...",
                  #   metadata={"title": "Authentication"},
                  #   rerank_score=None,
                  # )
```
</CodeGroup>

## With reranking

Set `use_rerank=True`. The reranker is created from the constructor `reranker` config (or pass one explicitly). Retrieve wide, keep the best few.

```python
result = rag.retrieve(
    "How do I authenticate?",
    top_k=20,
    use_rerank=True,
    rerank_top_k=5,
)
```

## With query rewriting

Turn a conversational follow-up into a standalone query before searching. The rewrite LLM falls back to the constructor `llm`.

```python
result = rag.retrieve(
    "What about the second step?",
    use_query_rewrite=True,
    session_context="User is asking about the onboarding flow.",
)

result.search_query  # the rewritten query that was searched
```

## `retrieve()` parameters

| Parameter | Default | Description |
| --- | --- | --- |
| `query` | required | Search query string |
| `embedder` | inherits constructor (`input_type="query"`) | Query embedder; auto-created when omitted |
| `vector_db` | inherits constructor | `qdrant`, `qdrant_cloud`, `pinecone`, `weaviate` |
| `collection` | inherits constructor | Collection name |
| `url` | inherits constructor | Vector DB URL |
| `top_k` | `10` | Candidates to retrieve |
| `use_query_rewrite` | `False` | Rewrite the query before searching |
| `rewrite_llm` | inherits constructor `llm` | LLM used for rewriting |
| `rewrite_context` / `session_context` | `""` | Context hints for the rewriter |
| `use_rerank` | `False` | Rerank results after retrieval |
| `reranker` | inherits constructor | Reranker object; auto-created when omitted |
| `rerank_top_k` | `5` | Results to keep after reranking |

## `reranker()` parameters

| Parameter | Default | Description |
| --- | --- | --- |
| `provider` | constructor default, else `"voyage"` | `"voyage"` (API) or `"bm25"` (local) |
| `model` | constructor default, else `"rerank-2-lite"` | Voyage: `rerank-2-lite`, `rerank-2`; BM25: `bm25` |
| `options` | `None` | Provider-specific options |

<Note>`bm25` is a local, keyword-based reranker with no API key. Use Voyage when meaning matters; BM25 for offline or cost-sensitive setups. Register others with `register_reranker()`.</Note>
