---
title: "REST API Overview"
description: "Run Ragrails behind an HTTP API."
---

The REST API server is best for language-agnostic applications and hosted integrations.

```bash
ragrails-api
```

The API listens on `http://127.0.0.1:8000` by default.

```bash
ragrails-api --host 0.0.0.0 --port 8080
```

Swagger UI and OpenAPI are available at:

```text
http://127.0.0.1:8000/docs
http://127.0.0.1:8000/v1/openapi.json
```

Use Swagger UI at `/docs` to inspect request schemas, try endpoints from the
browser, and copy generated curl examples.

| Stage | Endpoint | Details |
| --- | --- | --- |
| Health | `GET /v1/health` | This page |
| Ingestion | `POST /v1/ingest/api`, `/v1/ingest/url`, `/v1/ingest/docs` | [Ingestion](/usage/server/ingestion) |
| Chunking | `POST /v1/chunk` | [Chunking](/usage/server/chunking) |
| Embedding | `POST /v1/embed` | [Embedding](/usage/server/embedding) |
| Storing | `POST /v1/store` | [Storing](/usage/server/storing) |
| Retrieval | `POST /v1/retrieve` | [Retrieval](/usage/server/retrieval) |
