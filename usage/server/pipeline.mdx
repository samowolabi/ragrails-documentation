Two endpoints run multiple stages in a single request, the same as the SDK [`ingest()` and `query()`](/usage/sdk/pipeline).

## Ingest (`POST /v1/pipelines/ingest`)
Runs ingestion, chunking, embedding, and storage. Combine `docs`, `urls`, `api`, and `markdown` in one call.

Set `"concurrency": "parallel"` to run independent `docs`, `urls`, and `api` source ingestion together before chunking. The default is `serial`.

<CodeGroup>
```bash Request
curl -X POST http://127.0.0.1:8000/v1/pipelines/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "docs": {"folder": "files/docs/"},
    "urls": ["https://example.com/docs"],
    "chunking": {"chunk_size": 1000, "chunk_overlap": 100},
    "embedding": {"provider": "voyage", "model": "voyage-3"},
    "storage": {"vector_db": "qdrant", "collection": "docs", "url": "http://localhost:6333"},
    "concurrency": "parallel"
  }'
```

```json Response
{ "sources": 8, "chunks": 64, "embedded": 64, "stored": 64, "failed": 0, "errors": [] }
```
</CodeGroup>

| Field | Description |
| --- | --- |
| `docs` | Document files or folder, passed to parse |
| `urls` | URLs to scrape |
| `api` | API endpoint config to fetch |
| `markdown` | Raw markdown documents |
| `ingestion` | Ingestor config overrides |
| `chunking` | Chunking config, e.g. `{"chunk_size": 1000}` |
| `embedding` | Embedding config, e.g. `{"provider": "voyage", "model": "voyage-3"}` |
| `storage` | Storage config, e.g. `{"vector_db": "qdrant", "collection": "docs", "url": "..."}` |
| `concurrency` | Source ingestion mode: `serial` or `parallel` |

## Query (`POST /v1/pipelines/query`)
Runs query embedding and retrieval. The `retrieval` block accepts all retrieve options plus `rerank` and `query_rewrite` sub-objects.

<CodeGroup>
```bash Request
curl -X POST http://127.0.0.1:8000/v1/pipelines/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What does the guide cover?",
    "embedding": {"provider": "voyage", "model": "voyage-3"},
    "retrieval": {
      "vector_db": "qdrant",
      "collection": "docs",
      "url": "http://localhost:6333",
      "top_k": 20,
      "rerank": {"enabled": true, "provider": "voyage", "top_k": 5}
    }
  }'
```

```json Response
{
  "query": "What does the guide cover?",
  "search_query": "What does the guide cover?",
  "retrieved": 5,
  "items": [ { "id": "guide-7", "score": 0.83, "text": "The guide covers setup, auth, and usage...", "metadata": {}, "rerank_score": 0.91 } ],
  "failed": 0,
  "errors": []
}
```
</CodeGroup>

| Field | Description |
| --- | --- |
| `query` | Search query |
| `embedding` | Embedding config (all embedder params) |
| `retrieval` | Retrieval config: connection, `top_k`, plus `query_rewrite` and `rerank` sub-objects |
