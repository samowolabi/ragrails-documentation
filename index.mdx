title: "Ragrails"
description: "A developer toolkit for turning websites, documents, and APIs into retrieval-ready knowledge bases."
---

Ragrails is an end-to-end RAG preparation toolkit. It ingests source content,
converts it to markdown, chunks it with structure-aware metadata, embeds and
stores the chunks, then retrieves relevant context for applications and agents.

## What Ragrails does

| Capability | What you get |
| --- | --- |
| Website ingestion | Scrape exact URLs, crawl full sites, write markdown, and retry failed URL jobs from a dead-letter queue. |
| Document ingestion | Parse PDFs, DOCX files, spreadsheets, CSV/TSV, Markdown, HTML, JSON, XML, ZIP files, and other supported local formats. |
| REST API ingestion | Fetch API responses with methods, headers, query params, JSON bodies, pagination, and max page limits. |
| Structure-aware chunking | Split markdown while preserving headings, code blocks, tables, links, stable chunk IDs, and useful metadata. |
| Embedding and storage | Batch chunks through an embedding provider and write vectors to Qdrant, Pinecone, or Weaviate. |
| Retrieval and reranking | Search stored chunks with top-k retrieval and optional reranking. |
| Multiple interfaces | Use the same pipeline through the Python SDK, CLI, REST API server, or local MCP server. |
