`chat()` takes controls for conversation flow and low-quality retrieval. Each targets a specific failure mode.

## Conversation management

<CodeGroup>
```python SDK
from ragrails import HistoryCompactionConfig, IntentRoutingConfig

rag.chat(
    query,
    history=history,
    history_compaction=HistoryCompactionConfig(enabled=True, history_limit=15, keep_recent=5),
    intent_routing=IntentRoutingConfig(enabled=True),
)
```

```bash CLI
ragrails chat "$QUERY" \
  --history-file files/chat-history.json \
  --history-compaction \
  --intent-routing \
  --vector-db qdrant --collection docs --url http://localhost:6333 \
  --embedder-provider voyage --embedder-model voyage-3 \
  --llm-provider openai --llm-model gpt-4o-mini
```

```bash REST API
curl -X POST http://127.0.0.1:8000/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"query": "How do I authenticate?", "history_compaction": {"enabled": true, "history_limit": 15, "keep_recent": 5}, "intent_routing": {"enabled": true}, "llm_provider": "openai", "llm_model": "gpt-4o-mini", "embedder_provider": "voyage", "embedder_model": "voyage-3", "vector_db": "qdrant", "collection": "docs", "url": "http://localhost:6333", "history": []}'
```
</CodeGroup>

## Answer quality

Sometimes your index simply doesn't contain the answer. Rather than hallucinate, Ragrails scores the retrieved context and acts on low confidence.

<CodeGroup>
```python SDK
from ragrails import ChatRetrievalQualityConfig

rag.chat(
    query,
    history=history,
    retrieval_quality=ChatRetrievalQualityConfig(
        min_retrieval_score=0.35,
        min_rerank_score=0.50,
        low_confidence_mode="answer_with_caution",
    ),
)
```

```bash CLI
ragrails chat "$QUERY" \
  --vector-db qdrant --collection docs --url http://localhost:6333 \
  --embedder-provider voyage --embedder-model voyage-3 \
  --llm-provider openai --llm-model gpt-4o-mini
```

```bash REST API
curl -X POST http://127.0.0.1:8000/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"query": "Can I get a refund after 30 days?", "retrieval_quality": {"min_retrieval_score": 0.35, "min_rerank_score": 0.50, "low_confidence_mode": "answer_with_caution"}, "llm_provider": "openai", "llm_model": "gpt-4o-mini", "embedder_provider": "voyage", "embedder_model": "voyage-3", "vector_db": "qdrant", "collection": "docs", "url": "http://localhost:6333", "history": []}'
```
</CodeGroup>

<Note>The CLI does not expose retrieval-quality threshold flags yet; use SDK or REST for those controls.</Note>

| `low_confidence_mode` | Behaviour when context is weak |
| --- | --- |
| `answer_with_caution` | Answers, but flags the uncertainty (default) |
| `ask_clarifying_question` | Asks the user to refine the question instead of answering |
| `refuse_grounded_answer` | Declines to answer rather than risk a wrong one |

Every response carries `answer_confidence` and `retrieval_quality`. Surface them in your UI to flag shaky answers.
