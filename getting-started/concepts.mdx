Ragrails has four usage surfaces over the same staged pipeline.

| Surface | Positioning |
| --- | --- |
| SDK | Python-first application development. |
| CLI | Terminal workflows, local automation, and CI tasks. |
| REST API | HTTP integrations from any language or hosted service. |
| MCP | AI clients and agents that need local RAG tools. |

## Stage contract

Each stage consumes the previous stage's output:

| Stage | Input | Output |
| --- | --- | --- |
| Ingestion | URL, document files, or REST API response | Markdown files with metadata |
| Chunking | Markdown files | JSON chunk files |
| Embedding | JSON chunk files | Vectors generated by an embedder |
| Storing | Chunks plus vectors | Records in a vector database |
| Retrieval | Query text | Ranked result chunks |

Use the feature pages for the main workflow docs. Each feature page has SDK, CLI, REST API, and MCP tabs so you can stay focused on the RAG stage while switching implementation surface.
