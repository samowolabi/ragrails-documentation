title: "Configuration"
description: "Configure providers, paths, and local runtime behavior."
---

Ragrails favors explicit arguments. Most examples use local `files/input` and `files/output` folders, but every stage accepts path options.

## Common environment variables

Set provider credentials in your shell or `.env` file as needed:

```bash
VOYAGE_API_KEY=...
QDRANT_URL=http://localhost:6333
QDRANT_API_KEY=...
PINECONE_API_KEY=...
WEAVIATE_URL=...
WEAVIATE_API_KEY=...
OPENAI_API_KEY=...
ANTHROPIC_API_KEY=...
```

## Common paths

| Path | Purpose |
| --- | --- |
| `files/input` | Local source documents. |
| `files/output/web_crawled` | URL ingestion markdown output. |
| `files/output/docs` | Document ingestion markdown output. |
| `files/output/api` | API ingestion markdown output. |
| `files/output/chunks` | Chunk JSON output. |

## Provider defaults

Embedding and retrieval default to `embedder="voyage"` and `model="voyage-3"`. Vector database examples use Qdrant, but Pinecone and Weaviate are supported through the same stage arguments.
