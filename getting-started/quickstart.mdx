This example builds a small support knowledge base, then chats over it. Run it as-is, then swap in your own content.

## 1. Install and start Qdrant

```bash
pip install "ragrails[voyage,qdrant]"
export VOYAGE_API_KEY="..."
export OPENAI_API_KEY="..."
docker run -p 6333:6333 qdrant/qdrant
```

<Note>
LLM providers (OpenAI, Anthropic, Google) are built in, so there's no LLM extra.

This quickstart uses `openai` with `gpt-4o-mini`.
</Note>

<Tip>
See [models](/reference/models) for other supported LLM providers and model names, [vector databases](/reference/vector-databases) for supported vector stores, and [environment variables](/getting-started/env-examples) for provider keys.
</Tip>

## 2. Build and chat

Each example creates the same `support` collection from the same refund-policy text, runs one retrieval query, then runs one chat turn.

<CodeGroup>
```python SDK
from ragrails import RagRails

rag = RagRails(
    collection="support",
    vector_store={"provider": "qdrant", "url": "http://localhost:6333"},
    embedding={"provider": "voyage", "model": "voyage-3"},
    llm={"provider": "openai", "model": "gpt-4o-mini"},
)

rag.ingest(docs=["files/refund-policy.pdf"])

# Other source forms use the same pipeline:
# rag.ingest(urls="https://example.com/help/refunds")  # Scrape Website
# rag.ingest(api={"url": "https://api.example.com/refund-policy"})  # Fetch API responses
# rag.ingest(markdown="# Refund policy\n\nCustomers can request a refund within 30 days of purchase.")  # Pass Markdown directly

# Query retrieves matching chunks from the collection.
query_result = rag.query("How long do I have to request a refund?", retrieval={"top_k": 5})
print(query_result.items)

# Chat retrieves context and asks the configured LLM for a grounded answer.
chat_result = rag.chat("How long do I have to request a refund?", history=[])
print(chat_result.answer)
```

```bash CLI
ragrails ingest \
  --markdown "# Refund policy\n\nCustomers can request a refund within 30 days of purchase. Refunds are returned to the original payment method within 5 business days." \
  --vector-db qdrant --collection support --url http://localhost:6333 \
  --provider voyage --model voyage-3

ragrails query "How long do I have to request a refund?" \
  --vector-db qdrant --collection support --url http://localhost:6333 \
  --provider voyage --model voyage-3 --top-k 5

ragrails chat "How long do I have to request a refund?" \
  --vector-db qdrant --collection support --url http://localhost:6333 \
  --embedder-provider voyage --embedder-model voyage-3 \
  --llm-provider openai --llm-model gpt-4o-mini
```

```bash REST API
curl -X POST http://127.0.0.1:8000/v1/pipelines/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "markdown": "# Refund policy\n\nCustomers can request a refund within 30 days of purchase. Refunds are returned to the original payment method within 5 business days.",
    "embedding": {"provider": "voyage", "model": "voyage-3"},
    "storage": {"vector_db": "qdrant", "collection": "support", "url": "http://localhost:6333"}
  }'

curl -X POST http://127.0.0.1:8000/v1/pipelines/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "How long do I have to request a refund?",
    "embedding": {"provider": "voyage", "model": "voyage-3"},
    "retrieval": {"vector_db": "qdrant", "collection": "support", "url": "http://localhost:6333", "top_k": 5}
  }'

curl -X POST http://127.0.0.1:8000/v1/chat \
  -H "Content-Type: application/json" \
  -d '{
    "query": "How long do I have to request a refund?",
    "llm_provider": "openai",
    "llm_model": "gpt-4o-mini",
    "embedder_provider": "voyage",
    "embedder_model": "voyage-3",
    "vector_db": "qdrant",
    "collection": "support",
    "url": "http://localhost:6333",
    "history": []
  }'
```
</CodeGroup>

<Tip>
SDK defaults live in `RagRails(...)`.

CLI defaults live in `.ragrails.toml` or explicit flags.

REST defaults live in request payload fields.

Chat is stateless: save the returned history and pass it to the next turn.
</Tip>
